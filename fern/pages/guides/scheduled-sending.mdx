AgentMail's scheduled sending lets you create emails now and have them sent automatically at a future time. Combined with `Webhooks` and `Labels`, you can build conditional workflows like "follow up if no reply in 3 days."

## Schedule Send Basics

To schedule an email, create a `Draft` with the `send_at` field set to an ISO 8601 datetime. AgentMail automatically sends it at the specified time.

<CodeBlocks>
```python title="Python"
from datetime import datetime, timedelta

send_time = (datetime.utcnow() + timedelta(days=1)).replace(hour=9, minute=0, second=0)

draft = client.inboxes.drafts.create(
    inbox_id="outreach@domain.com",
    to=["prospect@example.com"],
    subject="Following up on our conversation",
    text="Hi, just wanted to follow up...",
    send_at=send_time.isoformat() + "Z"
)

print(f"Scheduled for {draft.send_at}")
print(f"Status: {draft.send_status}")  # "scheduled"
```

```typescript title="TypeScript"
const sendTime = new Date();
sendTime.setDate(sendTime.getDate() + 1);
sendTime.setUTCHours(9, 0, 0, 0);

const draft = await client.inboxes.drafts.create(
    "outreach@domain.com",
    {
        to: ["prospect@example.com"],
        subject: "Following up on our conversation",
        text: "Hi, just wanted to follow up...",
        sendAt: sendTime.toISOString()
    }
);

console.log(`Scheduled for ${draft.sendAt}`);
console.log(`Status: ${draft.sendStatus}`);  // "scheduled"
```
</CodeBlocks>

When you set `send_at`:
- The `Draft` is automatically labeled `scheduled`
- `send_status` is set to `scheduled`
- AgentMail sends the email at the specified time (polled every minute)
- After sending, the `Draft` is converted to a `Message` and deleted

## Conditional Follow-Ups

A common outreach pattern is: "Send a follow-up email in 3 days, but only if the prospect hasn't replied." AgentMail doesn't have a built-in conditional send, but you can implement this cleanly with `Drafts`, `Webhooks`, and `Labels`.

### How It Works

```
1. Send initial email
2. Create a scheduled follow-up Draft for 3 days later
3. Store the follow-up draft_id (label it on the thread)
4. Listen for replies via Webhook
5. If a reply comes in → delete the follow-up Draft
6. If no reply → Draft sends automatically at the scheduled time
```

### Step-by-Step Implementation

<Steps>
    <Step title="1. Send the Initial Email and Schedule a Follow-Up">
        Send your first email, then immediately create a scheduled `Draft` as the follow-up. Label the `Thread` so you can link the follow-up draft to it later.

        <CodeBlocks>
        ```python title="Python"
        from datetime import datetime, timedelta

        inbox_id = "outreach@domain.com"
        prospect = "prospect@example.com"

        # Send the initial email
        initial = client.inboxes.messages.send(
            inbox_id=inbox_id,
            to=[prospect],
            subject="Quick question about your workflow",
            text="Hi, I noticed your team is scaling quickly...",
            labels=["outreach", "awaiting-reply"]
        )

        # Schedule a follow-up draft for 3 days later
        follow_up_time = (datetime.utcnow() + timedelta(days=3)).replace(hour=9, minute=0, second=0)

        follow_up = client.inboxes.drafts.create(
            inbox_id=inbox_id,
            to=[prospect],
            subject="Re: Quick question about your workflow",
            text="Hi again — just bumping this in case it got buried...",
            in_reply_to=initial.message_id,
            send_at=follow_up_time.isoformat() + "Z"
        )

        # Label the thread with the follow-up draft ID so we can find it later
        client.inboxes.threads.update(
            inbox_id=inbox_id,
            thread_id=initial.thread_id,
            add_labels=[f"follow-up:{follow_up.draft_id}"]
        )
        ```

        ```typescript title="TypeScript"
        const inboxId = "outreach@domain.com";
        const prospect = "prospect@example.com";

        // Send the initial email
        const initial = await client.inboxes.messages.send(
            inboxId,
            {
                to: [prospect],
                subject: "Quick question about your workflow",
                text: "Hi, I noticed your team is scaling quickly...",
                labels: ["outreach", "awaiting-reply"]
            }
        );

        // Schedule a follow-up draft for 3 days later
        const followUpTime = new Date();
        followUpTime.setDate(followUpTime.getDate() + 3);
        followUpTime.setUTCHours(9, 0, 0, 0);

        const followUp = await client.inboxes.drafts.create(
            inboxId,
            {
                to: [prospect],
                subject: "Re: Quick question about your workflow",
                text: "Hi again — just bumping this in case it got buried...",
                inReplyTo: initial.messageId,
                sendAt: followUpTime.toISOString()
            }
        );

        // Label the thread with the follow-up draft ID so we can find it later
        await client.inboxes.threads.update(
            inboxId,
            initial.threadId,
            { addLabels: [`follow-up:${followUp.draftId}`] }
        );
        ```
        </CodeBlocks>
    </Step>

    <Step title="2. Listen for Replies via Webhook">
        Set up a `Webhook` to listen for `message.received` events. When a reply comes in on a thread that has a pending follow-up, cancel the scheduled `Draft`.

        <CodeBlocks>
        ```python title="Python"
        @app.route('/webhook/agentmail', methods=['POST'])
        def handle_webhook():
            payload = request.json

            if payload.get('type') != 'message.received':
                return Response(status=200)

            message = payload['message']
            inbox_id = message['inbox_id']
            thread_id = message['thread_id']

            # Get the thread to check for a pending follow-up
            thread = client.inboxes.threads.get(
                inbox_id=inbox_id,
                thread_id=thread_id
            )

            # Look for a follow-up label like "follow-up:<draft_id>"
            for label in thread.labels:
                if label.startswith("follow-up:"):
                    draft_id = label.split("follow-up:")[1]

                    # Cancel the scheduled follow-up
                    try:
                        client.inboxes.drafts.delete(
                            inbox_id=inbox_id,
                            draft_id=draft_id
                        )
                        print(f"Cancelled follow-up {draft_id} — prospect replied!")
                    except Exception:
                        pass  # Draft may have already been sent

                    # Clean up the label
                    client.inboxes.threads.update(
                        inbox_id=inbox_id,
                        thread_id=thread_id,
                        remove_labels=[label, "awaiting-reply"],
                        add_labels=["replied"]
                    )
                    break

            return Response(status=200)
        ```

        ```typescript title="TypeScript"
        app.post('/webhook/agentmail', async (req, res) => {
            const payload = req.body;

            if (payload.type !== 'message.received') {
                return res.sendStatus(200);
            }

            const { inbox_id: inboxId, thread_id: threadId } = payload.message;

            // Get the thread to check for a pending follow-up
            const thread = await client.inboxes.threads.get(inboxId, threadId);

            // Look for a follow-up label like "follow-up:<draft_id>"
            for (const label of thread.labels) {
                if (label.startsWith("follow-up:")) {
                    const draftId = label.split("follow-up:")[1];

                    // Cancel the scheduled follow-up
                    try {
                        await client.inboxes.drafts.delete(inboxId, draftId);
                        console.log(`Cancelled follow-up ${draftId} — prospect replied!`);
                    } catch {
                        // Draft may have already been sent
                    }

                    // Clean up the label
                    await client.inboxes.threads.update(inboxId, threadId, {
                        removeLabels: [label, "awaiting-reply"],
                        addLabels: ["replied"]
                    });
                    break;
                }
            }

            return res.sendStatus(200);
        });
        ```
        </CodeBlocks>
    </Step>
</Steps>

### What Happens

| Scenario | Result |
|----------|--------|
| Prospect replies within 3 days | Webhook fires, follow-up draft is deleted, no duplicate email sent |
| Prospect doesn't reply | Follow-up draft sends automatically at the scheduled time |
| You want to change the follow-up | Update the draft's `send_at` or content before it sends |

<Callout intent="success" title="Scaling This Pattern">
This pattern works for multi-step sequences too. After the follow-up sends, your webhook can detect the `message.sent` event and schedule a second follow-up — building a full drip campaign with conditional logic at each step.
</Callout>
